## Accessibility

This textarea component is built on the USWDS textarea and satisfies the following WCAG 2.1 AA criteria:

- **1.3.1 Info and Relationships** — Uses a native `<textarea>` element associated with a `<label>` via matching `id` and `htmlFor` attributes
- **1.4.3 Contrast (Minimum)** — Meets a 4.5:1 contrast ratio between text and background in all states
- **2.1.1 Keyboard** — Fully operable via keyboard — focusable, typeable, and navigable without a mouse
- **2.4.7 Focus Visible** — Displays a visible focus indicator when the textarea receives keyboard focus
- **3.3.1 Error Identification** — When in an error state, the error is communicated via text and `aria-invalid` — not color alone
- **3.3.2 Labels or Instructions** — Every textarea has a persistent visible label that does not disappear on focus

## Usage Guidelines

- Use a textarea for open-ended, multi-line input — comments, descriptions, feedback, notes
- Do not use a textarea when the expected input is a single line — use a TextInput instead
- Always pair a textarea with a Label component — never rely on placeholder text as a substitute
- Consider adding a character count indicator for textareas with a maximum length requirement — this helps users manage their input before hitting a limit
- Size the textarea to reflect the expected length of input — a comments field should be taller than a short description field
- Place the label directly above the textarea, not beside or below it
