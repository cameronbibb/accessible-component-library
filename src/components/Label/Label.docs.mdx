## Accessibility

This label component is built on the USWDS label and satisfies the following WCAG 2.1 AA criteria:

- **1.3.1 Info and Relationships** — Uses a native `<label>` element with an `htmlFor` attribute that associates it with its input by matching the input's `id`
- **2.4.6 Headings and Labels** — Labels are descriptive and clearly identify the purpose of the associated input
- **3.3.2 Labels or Instructions** — Every form input has a visible label that persists regardless of input state

## Usage Guidelines

- Every form input must have an associated Label — never rely on placeholder text as a substitute for a label
- The `htmlFor` prop must exactly match the `id` of the associated input — this is what connects the label to the input for screen readers
- Use the `requiredMarker` prop on fields that are required — this adds a visual indicator and communicates the requirement clearly
- Keep label text short and descriptive — it should clearly identify what the field expects
- Do not use the same label text for two different inputs on the same form
- Labels should be placed directly above their associated input, not beside or below it
